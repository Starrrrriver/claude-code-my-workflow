# papers/ — HEP Manuscript Projects

This directory holds all active and completed paper projects.
Each paper lives in its own subdirectory following the conventions below.

---

## Folder Convention

```
papers/
└── [paper-slug]/                        # e.g., bsm-dijets-atlas-2025
    ├── README.md                        # Title, journal, manuscript #, status
    ├── manuscript/                      # Source .tex files (main.tex, macros.tex, refs.bib)
    │   ├── main.tex
    │   ├── macros.tex                   # (optional) custom macros
    │   └── refs.bib
    ├── reviewer_comments/               # Raw referee reports
    │   ├── round1/
    │   │   ├── editor_letter.txt
    │   │   ├── reviewer1.txt
    │   │   └── reviewer2.txt
    │   └── round2/                      # (if applicable)
    └── responses/                       # Generated response documents
        ├── response_draft.md            # Markdown draft (generated by /reviewer-response)
        ├── response_letter.tex          # LaTeX letter (generated on request)
        └── hep_review_YYYY-MM-DD.md    # HEP domain reviewer report
```

---

## Naming Conventions

Paper slugs must be **lowercase, hyphenated, with year suffix**:

| Pattern | Example |
|---------|---------|
| `[topic]-[experiment]-[year]` | `bsm-dijets-atlas-2025` |
| `[observable]-[process]-[year]` | `ht-multijet-cms-2024` |
| `[theory]-[channel]-[year]` | `susy-stop-pair-2025` |
| `[collaboration]-[paper-id]-[year]` | `cms-exo-23-001-2024` |

**Rules:**
- No spaces, no uppercase, no special characters except hyphens
- Always include the year (submission or publication year)
- Include the experiment/collaboration when disambiguation is needed
- Keep slugs short but meaningful (3-5 words max)

## Per-Paper README.md Format

Each paper subdirectory should have its own `README.md`:

```markdown
# [Paper Title]

**Journal:** [PRL / PRD / JHEP / PLB / EPJC / ...]
**Manuscript #:** [e.g., PRL-2025-12345]
**Authors:** [Collaboration or author list]
**arXiv:** [arXiv:XXXX.XXXXX]
**Status:** [Drafting / Submitted / Under Review / Revision / Accepted / Published]
**Submission date:** YYYY-MM-DD
**Revision deadline:** YYYY-MM-DD

## Abstract (one line)
[One-sentence summary of the result]

## Key Result
[e.g., "95% CL exclusion of gluino masses below 2.1 TeV in simplified model"]

## Round History
| Round | Sent | Reviewers | Points | Status |
|-------|------|-----------|--------|--------|
| 1 | YYYY-MM-DD | 2 | 7 | Response submitted |
```

## Quick Start: Drafting a Reviewer Response

### Step 1 — Create the paper directory

```bash
mkdir -p papers/bsm-dijets-atlas-2025/{manuscript,reviewer_comments/round1,responses}
```

### Step 2 — Add manuscript and reviewer comments

```bash
# Copy your .tex source
cp /path/to/main.tex papers/bsm-dijets-atlas-2025/manuscript/
cp /path/to/refs.bib papers/bsm-dijets-atlas-2025/manuscript/

# Copy reviewer comment files (plain text preferred; PDF also supported)
cp /path/to/reviewer1_report.txt papers/bsm-dijets-atlas-2025/reviewer_comments/round1/reviewer1.txt
cp /path/to/reviewer2_report.txt papers/bsm-dijets-atlas-2025/reviewer_comments/round1/reviewer2.txt
```

### Step 3 — Create the paper README

Create `papers/bsm-dijets-atlas-2025/README.md` with title, journal, manuscript number.

### Step 4 — Invoke the skill

```
/reviewer-response papers/bsm-dijets-atlas-2025
```

The skill will:
1. Validate the directory structure
2. Read and understand your manuscript
3. Parse all reviewer comments into a structured table
4. Ask you to confirm the parse before drafting
5. Generate `responses/response_draft.md`
6. Run the HEP domain reviewer agent
7. Optionally compile `responses/response_letter.tex`

---

## Notes

- **PDF comments:** If reviewer reports are only available as PDF, convert to `.txt` first
  (`pdftotext reviewer.pdf reviewer.txt`) for best parsing results.
- **Multiple rounds:** Keep each round's comments in `reviewer_comments/round1/`, `round2/`, etc.
  The skill defaults to the most recent round if multiple exist.
- **Confidentiality:** Reviewer comments are never committed to a public repo. Add paper directories
  to `.gitignore` if this is a public repository.
